# Data Directory Structure

This directory houses all data layers for the multi-parameter operational limits project. Detailed provenance and licensing information can be found in [provenance.md](./provenance.md).

## Layer Definitions

- **Raw/**: Regional derivatives of source data.
    - **AIS/**: European-filtered CSVs (46.5N-60.0N, -4.5E-15.0E). These are extracted from the 10GB+ DMA monthly archives to minimize local storage.
    - **Metocean/**: Raw NetCDF (NORA3) and ASCII (FINO) records.
    - **Operations/**: SCADA and structural logs.
- **Interim/**: Domain-specific extracts and cleaned time-series.
    - Includes `OM_Events_*.csv` (validated dwell events) and `Fleet_Registry_*.csv`.
    - Includes turbine coordinate files generated by `Scripts/prepare_turbine_data.py`.
- **Processed/**: Final synchronized 10-minute backbone training matrices. This layer represents the fusion of Logistics, Environment, and Operations.

## Data Format Discovery Notes
*To be populated during Phase 0 discovery.*

- **DMA AIS**: Expecting CSV with columns: `Timestamp`, `MMSI`, `Latitude`, `Longitude`, `SOG`, `COG`, `Heading`.
- **NORA3**: Expecting NetCDF (.nc) with variables: `hs`, `tp`, `thq`.
- **FINO1**: Expecting 10-minute averaged ASCII/CSV.
