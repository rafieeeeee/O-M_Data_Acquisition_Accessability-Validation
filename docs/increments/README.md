# Increment Specs

This directory holds short plan/spec documents for meaningful increments before
implementation starts. Specs are governed by `docs/governance.md`: each one
should define purpose, scope, assumptions, expected outputs, affected areas,
validation, documentation impact, review notes, and acceptance criteria.

## Active Specs

- [Fusion v2 evidence-readiness audit](fusion-v2-evidence-readiness-audit.md):
  report-only readiness audit to decide whether Fusion v2 can support later
  Stage 2 sensitivity comparisons.
